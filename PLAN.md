# Plan: flipper-agent-indicator

Generated by [`/plea`](https://plea.dev) | 2026-04-19 | depth: quick (6 questions + final check)

## Summary

Companion to `tmux-agent-indicator` that turns a Flipper Zero into a physical notification surface for AI agents (Claude/Codex/OpenCode). A macOS Python daemon (bleak) subscribes to the same hook events, streams a compact binary/JSON protocol over BLE Serial to a custom Flipper `.fap` running on Unleashed Firmware. The `.fap` renders per-agent text/LED/vibra patterns and sends ack frames back when the user presses OK on the device, which clears the indicator on both Flipper and the tmux plugin.

## Protocol (host ↔ Flipper over BLE Serial)

Compact framed messages. MTU-safe (< 100 bytes). One frame = one notification or one ack.

- **Host→Flipper `NOTIFY` frame**: `0x01 <agent_id:u8> <state:u8> <text_len:u8> <text:utf8…>`
  - `agent_id`: 0=claude, 1=codex, 2=opencode, 255=generic.
  - `state`: 0=off, 1=running, 2=needs-input, 3=done.
  - `text`: optional short label (e.g. tool name, permission summary); truncated to 64 chars.
- **Flipper→Host `ACK` frame**: `0x81 <agent_id:u8>` (sent when user presses OK on the notification screen).
- **Host→Flipper `PING` frame**: `0x02` (keepalive every 30s; Flipper replies with `0x82`).

Version byte reserved for future extension (bump via top nibble of tag byte when needed).

## Non-Functional Requirements

- **Auto-reconnect** (question #5): daemon must survive Flipper reboot/out-of-range; exponential backoff, no spam in logs.
- **Fast hook path** (question #1): CLI invocation must complete in < 50ms so it does not slow agent prompts. Achieved via UDS fire-and-forget to the long-lived daemon.
- **Firmware compatibility** (question #4): target Unleashed FW API version pinned in `application.fam`; README states supported range.
- **Fail-quiet** (question #1): if Flipper is unreachable, the hook and tmux-agent-indicator must still work. Daemon errors go to a log file, never stderr of the agent.

## Open Questions

- Concrete vibra/LED patterns per agent — defaulting to: Claude=blue/short buzz, Codex=green/double buzz, OpenCode=purple/long buzz; `needs-input` adds repeating triple-pulse, `done` adds single chirp. User can override in `config.toml`.
- Notification coalescing policy — single-event overwrite (latest wins, no queue) as the default; revisit if it feels lossy during real use.
- macOS BLE permissions flow — first run will trigger Bluetooth permission dialog for the daemon binary; may need LaunchAgent plist for unattended start.
- Persistent pairing — BLE pairing on Flipper typically requires the mobile-app handshake. Need to verify Unleashed exposes a pairing-less Serial profile, else add a first-time pairing step using the official Flipper iOS/Android app.

## Constraints

- macOS (Darwin 25.x) as the only supported host; bleak works on macOS via CoreBluetooth.
- Python ≥ 3.11 (asyncio TaskGroup, `tomllib`).
- ufbt + Unleashed FW toolchain required on the dev machine for firmware builds.
- No modifications to `tmux-agent-indicator` repo — this project is purely additive.
- No remote control of the agent itself (question #6). Back-channel is strictly a dismiss signal.
